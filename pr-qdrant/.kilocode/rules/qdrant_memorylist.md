# qdrant_memorylist.md

- **Проблема с /health**: Эндпоинт `/health` может отсутствовать в некоторых версиях Qdrant, что приводит к ошибке 404.
- **Правильная проверка**: Для проверки работоспособности надежнее всего обратиться к корневому эндпоинту (`/`). Успешный JSON-ответ подтверждает, что сервис запущен.
- **Диагностика**: При любых ошибках доступа к API (например, 404, нет ответа) первым делом нужно смотреть логи контейнера: `docker-compose logs qdrant`.
- **docker-compose**: Атрибут `version` в `docker-compose.yml` устарел и его можно удалить, чтобы избежать предупреждений.
- **Доступ**: Веб-интерфейс доступен по адресу `http://localhost:6333/dashboard`.
